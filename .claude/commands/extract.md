---
description: 최근 작업에서 학습 가능한 개념을 concepts/ 에 정리·보강
---

이번 세션·최근 작업에서 학습 가치 있는 개념을 `concepts/<분야>/<주제>.md`에 정리한다.

## 수행 단계

1. **세션 스캔** — 다룬 주제·등장 명령·해결한 문제·교정된 이해 추출
2. **개념 단위로 분리** — 한 노트 = 한 개념 원칙
3. **기존 노트 확인** — `concepts/` 안에 이미 있는지 → 있으면 보강, 없으면 신규
4. **양식 준수해서 작성**:

```markdown
# <개념명>

## 정의
한 줄.

## 왜 필요한가
없으면 어떤 일이 벌어지는가.

## 핵심 원리
내부 동작. 가능하면 다이어그램.

## 코드/명령 예시
최소 단위 (5-10줄).

## 흔한 오해
"사실은 ~" 형태로.

## 관련 개념
- [관련 노트](./related.md)

## 출처
- 등장 과제: B1-N
- 학습일: YYYY-MM-DD
- 참고 자료: URL
```

5. **INDEX 업데이트** — `concepts/<분야>/README.md`에 새 노트 링크 추가
6. **보고** — 작성한 노트 목록 + 보강한 노트 + 변경 요약

## 원칙

- 미사여구·중복·잡담 제거 — **알맹이만**
- 코드 예시는 학습 보조용 (과제 정답 X)
- 학생이 6개월 후 다시 봤을 때 "아 이거였지" 떠올릴 수 있게
- 잘못 알고 있던 것은 별도 섹션으로 강조

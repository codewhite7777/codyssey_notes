다음 주제를 학생 입장에서 단계적으로 깊이 있게 설명해줘: $ARGUMENTS

## 설명 구조 (반드시 이 순서)

1. **한 줄 정의** — 이게 무엇인가 (압축)
2. **왜 필요한가** — 어떤 문제를 풀기 위해 등장했는가, 없으면 어떤 일이 벌어지는가
3. **동작 원리** — 내부적으로 어떻게 작동하는가
   - 가능하면 ASCII 다이어그램 또는 단순 비유
   - 메모리 구조·실행 순서 등 시각화 가능한 건 시각화
4. **최소 예시** — 5-10줄 코드 또는 명령 (복잡한 실전 예 X, 본질 보여주기)
5. **흔한 오해 / 함정** — 학습자가 자주 잘못 아는 지점, "사실은 ~" 형태
6. **관련 개념** — `concepts/` 폴더에서 관련 노트 찾아 링크
7. **더 읽을 거리** — 공식 문서 1-2개 (URL은 정확한 것만)

## 원칙

- **즉시 코드 덤프 금지** — 개념 먼저
- 학생이 알 만한 비유나 이전 학습 개념과 연결
- 학생이 "왜?" 질문할 만한 지점 미리 짚기
- 한 번에 너무 많이 X — 핵심만, 깊이 있게
- 끝에 "이해 확인 질문" 1-2개 던지기 (선택)

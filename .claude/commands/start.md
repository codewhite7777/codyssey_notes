새 과제 작업 시작. 인자: $ARGUMENTS

예시:
- `/start B1-2 트러블슈팅`
- `/start B1-3 가계부`

## 수행 단계

1. **인자 파싱** — `<id>` 와 `<name>` 추출. 없으면 학생에게 물음
2. **디렉토리 확인** — `<id>/` 이미 있으면 학생에게 확인
3. **디렉토리 생성** — `mkdir <id>` (이미 있으면 그대로 사용)
4. **git init** — `cd <id> && git init` (독립 레포)
5. **기본 파일 생성**:
   - `README.md` — 과제 명세 자리 (학생이 채울 것, 양식 제공)
   - `.gitignore` — 학생에게 어떤 스택인지 묻고 적절한 템플릿
   - `evidence/` — 산출 증거 자료 (스샷·로그) 폴더
   - (선택) `CLAUDE.md` — 과제별 추가 컨벤션
6. **INDEX.md 업데이트** — 상태 🔵 → 🟡 (진행 중), 레포 경로 채우기
7. **학습 부트스트랩 안내**:
   - 먼저 명세를 README.md에 1페이지 요약 권장
   - 미지의 개념을 `concepts/`에 stub 만들기 권장
   - `/learn <개념>` 으로 학습부터 시작 권장
   - `/connect` 로 기존 학습 자산과 연결 확인

## README.md 양식

```markdown
# <과제 ID> <과제 이름>

## 명세 요약 (1페이지)
- 분야:
- 학습 시간:
- 핵심 산출물:

### 기능 요구
- ...

### 제약
- ...

## 학습 목표
- 이 과제를 통해 익혀야 할 핵심 개념 3-5개

## 진행 체크리스트
- [ ] 명세 정독
- [ ] 미지 개념 stub 생성
- [ ] 개념 학습 완료
- [ ] 설계 스케치
- [ ] 구현
- [ ] 검증·테스트
- [ ] 회고 + 개념 추출

## 회고 링크
- (작성 후 retrospectives/ 링크)
```

## 원격 GitHub 레포

원격 생성은 학생 의도 확인 후. 자동 생성 X.
학생이 원하면:
```bash
gh repo create codyssey-<id> --private --source=. --remote=origin --push
```

이번 세션에서 학생이 배운 내용을 정리하고 회고 노트 작성을 돕는다.

## 수행

1. **세션 학습 포인트 추출**:
   - 새로 익힌 개념
   - 막혔던 지점과 해결 방법
   - 잘못 알고 있던 부분 교정
   - 다음에 적용할 인사이트
2. **파일명 결정** — `retrospectives/YYYY-MM-DD-<주제>.md` (오늘 날짜)
3. **초안 작성**:

```markdown
# YYYY-MM-DD <주제>

## 오늘 배운 것
- 개념 1: 한 줄 요약
- 개념 2: 한 줄 요약

## 막혔던 지점
- 어떤 부분에서 막혔는가
- 왜 막혔는가 (지식 부족? 잘못된 가정?)

## 해결 과정
- 시도한 것 → 안 된 이유
- 결국 통한 방법

## 잘못 알고 있던 것
- 교정된 이해 (있으면)

## 다음에 적용할 것
- 다음 과제에서 활용할 인사이트
- 더 학습할 토픽

## 추출한 개념
- [개념명](../concepts/<분야>/<주제>.md)
- (없으면 `/extract` 추천)
```

4. **학생에게 보여주고 추가/수정 받기**
5. **개념 추출 추천** — 회고에서 언급된 개념 중 노트로 정리할 가치 있는 것 식별 → `/extract` 권장

## 원칙

- 자기 솔직한 회고를 유도 — "잘 했다"보다 "왜 막혔는지"가 자산
- 미래의 자신이 읽을 글이라는 관점
- 한 페이지 분량이면 충분 (장황한 회고는 안 읽힘)

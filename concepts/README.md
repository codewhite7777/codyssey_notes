# 학습 개념 인덱스

> 과제를 거치며 누적되는 학습 자산.
> 새 과제 시작 시 먼저 검토하고, 완료 시 `/extract`로 보강.

## 분야

### [linux/](./linux/)
시스템 권한, 네트워크, 프로세스, 파일시스템, 셸, cron, 사용자/그룹/ACL 등.

### [os/](./os/)
메모리 구조·계층, 스케줄링, 동시성, 데드락, 시그널, 프로세스/스레드 등.

### [python/](./python/)
데이터 모델, 제너레이터, 데코레이터, 타입 힌트, 모듈 시스템, dataclass, I/O 패턴 등.

### [git/](./git/)
브랜치 모델, PR/리뷰, 충돌 해결, reset/revert/stash, GitHub Flow, 협업 패턴 등.

## 노트 양식

```markdown
# <개념명>

## 정의
한 줄 정의.

## 왜 필요한가
어떤 문제를 풀기 위해 등장했는가. 없으면 어떤 일이 벌어지는가.

## 핵심 원리
내부 동작 원리. 가능하면 간단한 다이어그램.

## 코드/명령 예시
최소 단위 (5-10줄). 복사용이 아니라 이해 보조용.

## 흔한 오해
학습자가 자주 잘못 알거나 빠지는 함정.

## 관련 개념
- [관련 노트 1](./related-note.md)

## 출처
- 등장 과제: [B1-N](../../B1-N/)
- 참고 자료: 공식 문서 URL 등
```

## 가이드
- **한 노트 = 한 개념** (5분 안에 읽을 분량)
- 코드 예시는 **최소 단위** (이해 보조)
- 출처에 어느 과제·자료에서 등장했는지 명시
- 잘못된 이해를 발견하면 **수정 + 회고**에 메모

# 파일 권한 = 방마다 붙은 출입 표찰

## 한 마디로
모든 파일·폴더에는 **"이 방에 누가 들어와서 뭘 할 수 있는지"** 알리는 표찰이 붙어 있음.

## 왜 알아야 해?

- 내 일기장을 남이 못 보게
- 시스템 중요 파일을 일반 사용자가 못 망가뜨리게
- 실행할 수 있는 스크립트와 그렇지 않은 데이터 구분

이 과제에서:
- `monitor.sh`는 운영팀(agent-core)만 실행 가능하게
- `api_keys` 폴더는 운영팀만 들어가게
- 공유 업로드 폴더는 모든 주민이 쓸 수 있게

## 표찰의 구조 (3 그룹 × 3 권한)

각 파일에는 **9칸짜리 표찰**:

```
                    monitor.sh

   소유자(주인)       같은 모임          외부인
    r  w  x          r  w  x          r  w  x
    ─  ─  ─          ─  ─  ─          ─  ─  ─
    ✓  ✓  ✓          ✓  ✗  ✓          ✗  ✗  ✗
   읽기 쓰기 실행    읽기 쓰기 실행    읽기 쓰기 실행
```

→ "주인은 다 가능 / 같은 모임은 읽기·실행만 / 외부인은 아무것도 X"

### 3가지 권한 (rwx)

| 표시 | 파일에서 | 폴더에서 |
|---|---|---|
| `r` (read) | 내용 보기 | 안에 뭐 있는지 **목록** 보기 |
| `w` (write) | 내용 수정 | 안에 파일 **만들기/지우기** |
| `x` (execute) | **실행** (스크립트) | 안으로 **들어가기** (`cd`) |

★ **폴더에 `x` 없으면 들어갈 수도 없음**. 안의 파일 이름을 알아도 접근 X. (자주 헷갈림)

## 숫자로 표시: 8진법

각 칸을 숫자로 더해서 한 자리:
```
r = 4
w = 2
x = 1
─────
rwx = 7  (4+2+1)   다 가능
rw- = 6  (4+2)     읽기·쓰기 (실행 X)
r-x = 5  (4+1)     읽기·실행만
r-- = 4            읽기만
--- = 0            아무것도 X
```

`chmod 750 monitor.sh` 의미를 풀면:
```
7 5 0
│ │ └── 외부인:    ---  (아무것도 X)
│ └──── 같은 모임: r-x  (읽기·실행)
└────── 소유자:    rwx  (다)
```

## 한 번 보자

```bash
ls -l /etc/passwd
# 출력 예: -rw-r--r-- 1 root root 2123 ... /etc/passwd
#         └─┬┘└┬┘└┬┘ └┬┘ └┬┘
#           │  │  │   │   소유 모임
#           │  │  │   소유자 (root)
#           │  │  외부인 (r--)
#           │  같은 모임 (r--)
#           소유자 (rw-)
#         (맨 앞 - 는 일반 파일, d면 폴더, l이면 링크)

ls -l ~/.ssh/                  # 본인 SSH 폴더 (있으면)
ls -ld /tmp                    # 폴더 자체 권한 (-d 옵션이 핵심)
```

## 표찰 바꾸기 (chmod, chown)

```bash
chmod 750 monitor.sh                   # 8진법으로 한 번에
chmod u+x file                         # 소유자(u)에 실행(x) 추가
chmod g-w file                         # 모임에서 쓰기(w) 빼기
chmod o= file                          # 외부인(o) 다 없애기
chown agent-dev:agent-core monitor.sh  # 소유자:모임 둘 다 변경
```

## umask: 새 파일의 기본 표찰

새 파일을 만들면 기본 권한이 자동 부여됨. 이걸 `umask`가 결정:

```bash
umask 022
# 새 파일은 666 - 022 = 644 (rw-r--r--)
# 새 폴더는 777 - 022 = 755 (rwxr-xr-x)

umask 077
# 본인만 보고 쓸 수 있음 (보안 강한 환경)
```

## 자주 헷갈리는 것

- **폴더에 `r`만 있으면 안의 파일 접근 가능** → ❌ **`x` 필요**.
- **`chmod 777` 이 편함** → ❌ **누구나 망가뜨릴 수 있음 = 보안 재앙**.
- **내가 만든 파일이라 항상 권한 있다** → 본인이 권한 빼면 본인도 못 봄.
- **`chmod -R`로 폴더 권한 바꾸면 새 파일도 적용** → 아니. 새 파일은 `umask` 따름.

## 이번 과제(B1-1)에서

```
monitor.sh = 750 (rwxr-x---)
   소유자 (agent-dev): rwx     개발자가 수정
   모임 (agent-core):  r-x     운영자가 cron으로 실행 (admin이 core 멤버)
   외부인:             ---     agent-test 등은 못 봄

api_keys/ 폴더 = 모임 agent-core 만 접근 가능
   → 외부인 권한 0 + 모임에 rwx
```

> "한 폴더에 모임 두 개에 다른 권한" 같은 요구는 기본 권한으로 표현 불가 (모임은 1개뿐). → 다음에 ACL이라는 추가 도구 학습.

## 기술 용어 풀이

- **권한 비트(permission bit)** — 표찰의 한 칸 (r/w/x).
- **owner / group / other** — 소유자 / 같은 모임 / 외부인.
- **8진법(octal)** — 0~7로 권한을 묶어 표시하는 방식.
- **umask** — 새 파일의 기본 권한을 결정하는 마스크.
- **ACL** — 기본 표찰로 부족할 때 쓰는 "추가 표찰" (다음 학습).
